# Project Goals 
-  The goal of this project is to design and implement a bare-metal firmware for an environmental sensor on the STM32F401RE using a Bosch BME280 sensor. 
-  The project was built as a learning exercise in embedded systems fundementals, for this reason HAL has been avoided opting to use CMSIS driver headers with direct peripheral register access. 
-  This project demonstrates several core concepts : 
	-  GPIO config
	-  USART Tx bring up for debug
	-  I2C peripheral bring up and protocol handling
	-  EXTI configuration for external interrupt input 
-  The final behaviour of the system is : 
	-  The STM32 communicates with the BME280 over i2c1
	-  The BME280 is configured in normal mode with sampling rate of 1Hz
	-  Latest readings are maintained in software
	-  Pressing the user button (B1) triggers an EXTI interrupt
	-  The interrupt sets a flag, and the main loop responds by sending the latest readings to serial terminal via usart2
-  This project was designed not only to be a functioning application but also as a documented study project for developing understanding of peripheral behaviour and embedded software architecture.
