# Methodology 
## Design Approach 
-  This project was developed using an incremental bring up approach. Each module was implemented and tested independently before integrating into the wider system, with the intention to reduce complexity, isolate faults early and build understanding of each peripheral in a controlled way. 
-  This approach keeps the project focused on fundamental understanding rather than rushing a finished output. 
## Design principles
-  Bare-metal - this project intentionally avoided HAL abstraction and instead uses CMSIS drivers with direct register access. This forces fundemental understanding of peripheral and protocol function. 
-  Modules - deliberately separated functions into modules with clear roles, allowing easier understanding, testing and most importantly ease to extend project. 
-  Readability - Preferred readable logic and explicit state handling. 
## Development sequence
### Stage 1: Basic board bring-up
-  A minimal STM32 project was created with CMSIS support and startup/linker files. Basic clock assumptions were established so peripheral timing calculations could be made.
### Stage 2: USART debug output
-  USART2 was configured first so the board could send text to a host terminal through the ST-LINK virtual COM port. This provided a simple and reliable debugging interface for the rest of the project.
### Stage 3: I2C bring-up
-  I2C1 was configured for communication with the BME280. Initial testing focused on simple bus communication and chip ID reads before moving on to more complete register transactions.
### Stage 4: Sensor driver development
-  Once communication was verified, a BME280 driver was written to:
	-  Perform reset and chip identification
	-  Read calibration data
	-  Configure oversampling and mode settings
	-  Read raw measurement registers
### Stage 5: Compensation and application logic
-  Raw sensor values were converted into compensated temperature, pressure, and humidity readings using the Bosch compensation formulas. The application then maintained an up-to-date set of readings for later transmission.
### Stage 6: EXTI button input
-  The Nucleo user button was configured through EXTI so that a button press could trigger output of the latest readings. 
### Stage 7: Periodic update model
-  SysTick was used to create a simple periodic timing base. The main loop uses this to refresh sensor readings at a defined interval while remaining responsive to button-triggered serial output.
## Project outcome
-  By following this staged approach, the project became more than a simple sensor demo. It became a practical study in:
	-  STM32 peripheral bring-up
	-  Interrupt-driven driver design
	-  Module boundaries in embedded firmware
	-  Sensor interfacing at register level
	-  Integrating hardware events with application behaviour
-  The result is a small but complete bare-metal embedded application that is useful both as a working project and as a foundation for deeper embedded systems learning.
