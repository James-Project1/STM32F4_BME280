# Overview 
 -  The BME280 module, produces calibrated temperature, pressure and humidity readings printed on demand to the serial terminal triggered by a button press. BME280 module provides initialisation, raw data acquisition and compensated output for the Bosch BME280 sensor. Communication occurs over i2c, using the module previously described. 
 -  This module is split across two source files with a shared header : 
	 -  bme280_driver - all register-level I2C interaction. Owns reset, chip ID check, calibration reads, device config and raw ADC burst reads. 
	 -  bme280_processing - implements compensation formulas.

| Supply voltage    | 3.3V                               |
# Operating Modes 
-  BME280 supports three operating modes. 
	-  Sleep mode - no measurements, lowest power. Device remains in sleep. 
	-  Forced mode - Single measurement on demand. Device returns to sleep automatically after measurement completes. 
	-  Normal mode - autonomous continuous measurement at a set rate. 
-  Normal mode is used over forced mode to reduce latency of print time. The BME280 samples autonomously at 1Hz, this is then read by the STM32 over i2c and stored in an internal struct. When the button is pressed, the most recent reading will be printed. 
-  This approach was used over forced mode as it reduces print latency, do not need to request a measurement, then read the data over i2c, this is all done autonomously also it means have access to the readings for other tasks in future. However, normal mode is higher power. 
# Initialisation 
### Step 1 - Soft Reset
-  As explained in the BME280 RM, writing 0xB6 to register 0xE0 performs a device soft reset, guaranteeing the sensor starts from a know state regardless of prior condition. It is required that a blocking delay of at least 2ms is required after this write for device startup time, but for safety a 5ms blocking delay is used.
### Step 2 - Chip ID Verification 
-  Register 0xD0 returns the fixed chip ID, for the BME280 this is 0x60. This check ensures the device is found on the bus and returns the correct value, else register locations would not match. 
### Step 3 - Read Calibration Data
-  Read device specific coefficients used during the compensation formulas. Read at startup and stored internally. 
-  Note the coefficients are stored across two non-contiguous register banks, careful parsing of bytes is required to correctly extract coefficients. 
### Step 4 - Configure Device 
-  Three registers are written in this order : 

| Step | Register | Value |                   Purpose                    |
| :--: | :------: | :---: | :------------------------------------------: |
|  1   |   0xF2   | 0x01  |           Humidity oversampling x1           |
|  2   |   0xF4   | 0x27  |      Temp x1, pressure x1, normal mode       |
|  3   |   0xF5   | 0xA0  | Standby 1000ms (1Hz Sample), IIR filter off  |
-  Ordering is essential as stated in BME280 reference manual, it is a hardware constraint of ordering of steps 1 and 2. 
# Implementation 
### bme280_read_raw() 
-  This function is called in main.c every 1Hz by the measurement flag. 
-  The device is in normal mode, so the registers always contain a valid and completed measurement when this function runs. 
-  Ensure correct registers are read from, check the BME280 RM. 
### Compensation 
-  Must implement the Bosch integer compensation formulas to convert the raw ADC readings into calibrated physical values using the device specific coefficients read during initialisation. 
-  Integer compensation is used for simplicity and the additional accuracy of floating point is not required. (do not need to use FPU)
-  Will not go into further detail as the formulas are intuitive in the BME280 RM.
